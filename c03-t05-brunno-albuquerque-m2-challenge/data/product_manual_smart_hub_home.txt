================================================================================
OVERVIEW
================================================================================

The Smart Hub Home (TSP-SHH-200) is TechStore Plus's central IoT controller,
designed to unify devices across multiple smart home ecosystems. It supports
Zigbee 3.0, Z-Wave Plus, Wi-Fi 6, and Bluetooth 5.2 simultaneously, acting as
a protocol bridge so devices from different manufacturers can be automated
together through a single app.

Compatible ecosystems: Apple HomeKit, Google Home, Amazon Alexa, and the native
TechStore SmartHome app (iOS / Android). Local processing mode available —
automations run without a cloud connection after initial setup.

Key features:
- Controls up to 200 Zigbee/Z-Wave devices per hub
- Built-in Zigbee coordinator and Z-Wave controller
- Local automation engine (no cloud required for routines)
- Matter 1.2 bridge: expose Zigbee/Z-Wave devices to Matter controllers
- 2x USB-A ports for USB Zigbee/Z-Wave dongles (optional expansion)
- 1x Gigabit Ethernet port (recommended for stability)
- Mesh Wi-Fi extension when paired with Router NX300 or Router NX200

================================================================================
MESH NETWORKING REQUIREMENT
================================================================================

The Smart Hub Home's mesh Wi-Fi extension feature requires a Router NX300 or
Router NX200 as the primary network gateway. This is a REQUIRED dependency for
mesh functionality:

- The Smart Hub Home requires the Router NX300 (recommended) or Router NX200 to
  act as the mesh network controller. Without one of these routers, the Smart Hub
  Home functions only as a standalone smart device hub (Zigbee/Z-Wave controller)
  and cannot extend Wi-Fi coverage.
- When the Smart Hub Home is connected to the Router NX300 via Ethernet, together
  they cover up to 4,000 sq ft under a single unified SSID.
- The Router NX300 is preferred over the Router NX200 for mesh pairing because the
  NX300's Qualcomm IPQ5018 chipset supports Wi-Fi 6 mesh backhaul, delivering up
  to 40% better throughput in the extended coverage zone.

In summary: Smart Hub Home REQUIRES Router NX300 or Router NX200 for mesh
networking. The Smart Hub Home WORKS WITH Router NX300 as its primary mesh partner.

INITIAL SETUP
================================================================================

1. Place the Smart Hub Home near your Wi-Fi router for a stable connection.
   Wired Ethernet via the Router NX300 or Router NX200 is strongly recommended.
2. Connect the Ethernet cable from the hub to a LAN port on your router.
   NOTE: For mesh functionality, use the Router NX300 or Router NX200 only.
3. Connect the USB-C power adapter. The LED ring will pulse blue during boot
   (approximately 90 seconds).
4. Download the TechStore SmartHome app from the App Store or Google Play.
5. In the app, tap "Add Hub" and scan the QR code on the hub's bottom label.
6. Follow the in-app wizard:
   a. Name your home and location.
   b. Connect the hub to your Wi-Fi network (if not using Ethernet).
   c. Enable cloud sync if you want remote access and voice assistant integration.
   d. Accept the End-User License Agreement (EULA).
   e. If you have a Router NX300 or Router NX200, tap "Enable Mesh" to configure
      the Smart Hub Home as a secondary Wi-Fi access point.
7. Add your first device: tap "+" in the app and follow the device pairing flow
   for Zigbee, Z-Wave, or Wi-Fi devices.
8. Register your warranty at techstoreplus.com/warranty using the serial number
   on the bottom label (format: SHH-XXXXXXXX).

================================================================================
TROUBLESHOOTING
================================================================================

ISSUE: Hub LED pulses red — cannot connect to network
- Check that the Ethernet cable is firmly connected (preferred) or re-enter Wi-Fi
  credentials in the TechStore SmartHome app (Hub Settings > Network).
- Restart the hub (hold the side button 5 seconds until LED pulses white).
- Verify the router (Router NX300 or Router NX200) is online and operational.

ISSUE: Mesh Wi-Fi not working after setup
- Mesh requires the Router NX300 or Router NX200. If using a third-party router,
  mesh functionality is not available.
- Confirm the Smart Hub Home is connected to a LAN port (not WAN) on the router.
- In the NX300 app, go to Network > Mesh and verify the Smart Hub Home appears
  as an active mesh node. If not, re-run the mesh pairing from the NX300 app.

ISSUE: Zigbee device not pairing
- Put the hub into pairing mode (app > Hub > Add Device > Zigbee) and reset the
  target device according to its own manual.
- Ensure the device is within 15 meters of the hub during pairing.
- Update hub firmware to the latest version (app > Hub Settings > Firmware Update).

ISSUE: Smart Cam 4K motion alerts not triggering automations
- Verify the Smart Cam 4K is paired with the Smart Hub Home (app > Devices >
  Smart Cam 4K > Status: Paired).
- Check that the automation trigger is set to "Motion Detected — Smart Cam 4K"
  in the Automations editor.
- If the Smart Cam 4K is on the Router NX300 mesh network and the Smart Hub Home
  is the local hub, confirm both devices share the same network SSID.

ISSUE: Automations not triggering
- Check that Local Automation mode is enabled (app > Settings > Local Processing = ON).
- Verify that all devices involved in the automation are online (green status icon).
- Review the automation log (app > Automations > [name] > History) for errors.

ISSUE: App shows "Hub Offline"
- Check router firewall: the hub requires outbound access to hub.techstoreplus.com
  on port 443 for cloud features.
- If running in Local Mode only, the "Offline" banner is expected — local
  automations still run normally.
